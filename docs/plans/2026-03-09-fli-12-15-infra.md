# Implementation Plan: Docker Compose + Environment Configuration

## Issues
- FLI-12: Docker Compose setup
- FLI-15: Environment configuration

## Research Context

### Current State
- Python 3.12 project using `hatchling` build system (src-layout: `src/flight_watcher/`)
- Dependencies: `fast-flights` (from GitHub), `patchright` (needs `patchright install chromium` post-install)
- No Docker files, no `.env.example` exist yet
- `.gitignore` already covers `.env`, `.env.local`, `.env.*.local`
- No database code exists yet — this is the first infra step before FLI-13 (PostgreSQL schema)
- Entry point: `src/flight_watcher/__main__.py` with hardcoded routes
- Tests: pytest in `tests/` directory

### Architecture Constraints
- PostgreSQL 18-alpine per issue description
- Scanner service built from local Dockerfile
- Patchright requires Chromium browser installed in container
- Editable install (`pip install -e .`) for development workflow

## Decisions Made

**Base image:** `python:3.12-slim` — matches `.python-version`, slim keeps image small while providing build tools via apt.

**Chromium strategy:** Install system Chromium via apt (`chromium`), then set `PATCHRIGHT_BROWSERS_PATH` to use system browser. Avoids downloading Chromium inside container on every build. Fall back to `patchright install chromium` if system Chromium doesn't work with patchright.

**Compose profiles:** No profiles for now — both services always run together. Can add profiles later if needed.

**Network:** Default Compose network (implicit). No need for explicit network definition — all services in the same `docker-compose.yml` share a default network automatically.

**Volume strategy:** Named volume for PostgreSQL data persistence. Bind mount for source code in development (enables live reload).

**Health checks:** PostgreSQL via `pg_isready`. Scanner via a simple process check (no HTTP endpoint yet).

### Task 2: Create `Dockerfile` (FLI-12)
File: `Dockerfile`

Multi-stage not needed yet (no compiled assets). Single stage:
1. `python:3.12-slim` base
2. Install system deps: Chromium + required libs for headless browser
3. Set `PATCHRIGHT_BROWSERS_PATH=/usr/lib/chromium`
4. Copy `pyproject.toml` + `src/` (layer caching: deps first, code second)
5. `pip install .` (non-editable for production image)
6. Set entrypoint: `python -m flight_watcher`

### Task 4: Create `docker-compose.yml` (FLI-12)
File: `docker-compose.yml`

Services:
1. **db** — `postgres:18-alpine`
   - Environment from `.env` (POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD)
   - Named volume `pgdata:/var/lib/postgresql/data`
   - Health check: `pg_isready -U $POSTGRES_USER -d $POSTGRES_DB`
   - Port: `5432:5432` (for local dev access)

2. **scanner** — build from local `Dockerfile`
   - `depends_on: db` with `condition: service_healthy`
   - Environment from `.env` (all vars)
   - Bind mount `./src:/app/src:ro` for development (read-only)
   - Health check: process-based or skip (no HTTP server yet)

Volumes:
- `pgdata:` (named, for PostgreSQL persistence)

### Task 5: Create local `.env` from example
Copy `.env.example` to `.env` with development defaults. This file is gitignored so it won't be committed. Just ensure it exists for `docker compose up` to work.

**Do NOT commit `.env`** — only commit `.env.example`.

### Task 6: Verify Docker Compose validates
Run `docker compose config` to validate the compose file parses correctly. Do NOT run `docker compose up` (that requires Docker daemon and pulls images).
